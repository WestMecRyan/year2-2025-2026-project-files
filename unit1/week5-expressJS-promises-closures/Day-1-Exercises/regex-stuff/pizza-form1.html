Regex Form Validation Tutorial

🎓 Learning Objectives:

This tutorial demonstrates:

  • Regex Patterns: Using ^ (start), $ (end), character sets [a-z], quantifiers +, *, {}
  • String Methods: trim(), test(), match()
  • Error Handling: try-catch blocks for validation
  • Local Storage: Saving and retrieving form data
  • Real-time Validation: Instant feedback as you type
Personal Information
Regex: /^[A-Za-z]{2,30}$/
Meaning: ^ = start, [A-Za-z] = letters only, {2,30} = 2-30 characters, $ = end
Regex: /^[A-Za-z\s'-]{2,50}$/
Meaning: Allows letters, spaces, apostrophes, hyphens (for names like O'Connor, Van-Der)
Regex: /^[^\s@]+@[^\s@]+\.[^\s@]+$/
Meaning: [^\s@]+ = no spaces/@ symbols, @ = literal @, \. = literal dot
Regex: /^\(\d{3}\)\s\d{3}-\d{4}$/
Meaning: \( \) = literal parentheses, \d{3} = exactly 3 digits, \s = space
Address Information
Regex: /^\d+\s+[A-Za-z\s.,-]+$/g
Meaning: \d+ = one or more digits, \s+ = one or more spaces, g = global flag
Regex: /^[A-Za-z\s.-]{2,50}$/
Meaning: Letters, spaces, periods, hyphens (for cities like St. Louis)
Regex: /^[A-Z]{2}$/
Meaning: Exactly 2 uppercase letters
Regex: /^\d{5}(-\d{4})?$/
Meaning: \d{5} = 5 digits, (-\d{4})? = optional dash + 4 digits
